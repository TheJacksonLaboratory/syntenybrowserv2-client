Data Sources and Formats

Mouse Data

Data files from mouse source data. For ontologies pertaining to mouse see the General Data.

Name File Format Description
Mouse_GenomeFeatures_Synteny.gff3 GFF3
  • source: created from the file MGI.exome.gff3.gz, which can be found here
  • description: contains MGI's (Mouse Genome Informatics) genes and pseudogenes in minimized format (i.e. all exons combined into a single feature); each feature has its gene name, gene symbol, and MGI ID
Mouse_QTL_Synteny.gff3 GFF3
  • source: source data is taken from the MRK_List2.rpt file available on MGI's public FTP and then converted to GFF3
  • description: features in this file have the type QTL; the column 9 field contains 3 attributes - (1) the ID field is the feature's MGI ID, (2) the Name field is the QTL's official name, and (3) Dbxref is MGI's ID again but only the numeric portion
Mouse_GeneAssociation_MP_Synteny.txt GAF
  • source:
  • description: this file contains 4 columns; the first column is MGI's ID of the genome feature (MGI:1234); the second column is the feature symbol; the third column is the MP term for the specific annotation (MP:1234); the last column is the extended name for the specific MP term (increased systemic arterial diastolic blood pressure, etc)
Mouse_GeneAssocication_GO_Synteny.gaf GAF
  • source: source data is taken from the gene_association.mgi file available on MGI's public FTP and then converted to GAF
  • description: file's headers provide more complete description; however, the gene with annotation is stored in the second column (MGI:1234) and the specific annotation term is stored in the fourth column (GO:1234)
Mouse_CytoBand_synteny.gff3 GFF3
  • source: downloaded from UCSC
  • description: the original (UCSC) file has been converted into a standard GFF3 format and renamed for clarity; column 9 contains three attributes: ID (the provided UCSC ID), Location (the p/q arm position of this band), and Color (the stain color of this band)

Human Data

Data files from human source data. For ontologies pertaining to human see the general section.

Name File Format Description
Human_GenomeFeatures_Synteny.gff3 GFF3
  • source: this file contains all of NCBI's genes and pseudogenes in a minimized format (all exons combined into a single feature). It is created from the ref_GRCh38.p12_top_level.gff3 file from NCBI and contains the official gene symbol, gene name, and GeneID for each of these features.
  • description: this file is in standard GFF3 format with the following column 9 properties; ID, Symbol, Name, and Dbxref. Please note the NCBI GeneID is stored in the Dbxref field whereas the ID field should be used only to identify parent/child features within the GFF.
Human_GeneAnnotation_GO_Synteny.gaf GAF
  • source: source data is from a regularly generated report on the Gene Ontology Consortium's website and can be found here; the particular file to use is goa_human.gaf that is filtered to reduce complexity and circular annotations that exist in other files
  • description: the files headers provide a more complete description however the gene with annotation is stored in the second column (format A0A024RBG1) and the specific annotation term is stored in the 4th column (format GO:1234). In the downloaded version this ID is a UniProtKB ID but in the future it should be changed to a NCBI ID.
Human_GWAS_Synteny.vcf VCF
  • source: the source data is from the EBI GWAS database, in particular, 1.0.2 all associations with added ontology annotations
  • description: contains GWAS loci that match the term 'type II diabetes mellitus'; note that GWAS hits classified with multiple terms including type II diabetes mellitus were not included; this file was converted into a standard VCF format with 4 additional fields: AF (allele frequency), CG (mapped gene symbols), LT (ontology term for loci), and PMID (the study's PubMed ID)
Human_CytoBand_Synteny.gff3 GFF3
  • source: downloaded from UCSC
  • description: the original (UCSC) file has been converted into a standard GFF3 format and renamed for clarity; column 9 contains three attributes: ID (the provided UCSC ID), Location (the p/q arm position of this band), and Color (the stain color of this band)

Rat Data

Name File Format Description
Rat_GenomeFeatures_Synteny.gff3 GFF3
  • source: The files for rat were derived from RGD provided source files found here, in this case the gff3 file. In rat the top level feature is "gene."
  • description: This file contains all exons from the rn6 assembly for rat which is the most current genome assembly. This file is in standard GFF3 format with the following column 9 properties; ID, Symbol, Name, and Dbxref. Please note the MGI ID is stored in the Dbxref field whereas the ID field should be used only to identify parent/child features within the GFF.
Rat_QTL_Synteny.gff3 GFF3
  • source:
  • description:
Rat_GeneAssociation_GO_Synteny.gaf GAF
  • source: This file is in a standard GAF format with several comment lines (indicated by !) followed by the file data. This file is also a regularly generated report on RGD's website and can be found here. The file that is currently being used was provided by RGD in their Synteny Browser FTP folder but in the future the updated version should be used.
  • description: The files headers provide a more complete description however the gene with annotation is stored in the second column (just the numeric portion of the RGD ID) and the specific annotation term is stored in the 4th column (format GO:1234). In the downloaded version this ID is without its RGD prefix and should be changed in the future.
