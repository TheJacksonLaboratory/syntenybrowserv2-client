Data Sources and Formats

Mouse Data

Data files from mouse source data. For ontologies pertaining to mouse see the General Data section.

Name File Format Details
Mouse_GenomeFeatures_Synteny.gff3 GFF3
  • source: Mouse Genome Informatics - http://www.informatics.jax.org/
  • description: generated from the file MGI.exome.gff3.gz, which can be found here, it contains genes and pseudogenes in minimized format (i.e. all exons combined into a single feature). Each feature has a name, a symbol, and an MGI ID.
Mouse_QTL_Synteny.gff3 GFF3
  • source: Mouse Genome Informatics - http://www.informatics.jax.org/ - source data is taken from the MRK_List2.rpt file available on MGI's public FTP and then converted to GFF3 format
  • description: features in this file have the type QTL. The Column 9 field contains 3 attributes - 'ID' is the feature's MGI ID, 'Name' is the QTL's official name, and 'Dbxref' is MGI's ID again, but only the numeric portion.
Mouse_GeneAssociation_MP_Synteny.gaf GAF
  • source: release 22 Dec, 2020
  • description: a standard GAF format file with several comment lines (indicated by !) followed by the file data.
    Unlike a standard GAF file the gene ontology terms are instead mouse phenotype ID's. Several required fields for GO such as evidence codes, cardinality, etc are not included in this file because they do not apply to MP.
Mouse_GeneAssociation_GO_Synteny.gaf GAF
  • source: source data is taken from the gene_association.mgi file available on MGI's public FTP and then converted to GAF format
  • description: file's headers provide more complete description. The gene with annotation is stored in the second column (MGI:1234) and the specific annotation term is stored in the fourth column (GO:1234)
Mouse_GeneAssociation_DO_Synteny.gaf GAF
  • source: Mouse Genome Informatics - http://www.informatics.jax.org/downloads/reports/MGI_DO.rpt
  • description: standard GAF format file in which the feature with annotation is stored in the second column and the specific ontology annotation term is stored in the fourth column.
Mouse_CytoBand_synteny.gff3 GFF3
  • source: UCSC - https://www.ucsc.edu/ - release 12 Jun, 2020
  • description: the original (UCSC) file has been converted into standard GFF3 format and renamed.
    Column 9 contains 3 attributes: 'ID' - the provided UCSC ID, 'Location' - the p/q arm position of this band, and 'Color' - the stain color of this band.

Human Data

Data files from human source data. For ontologies pertaining to human see the General Data section.

Name File Format Details
Human_GenomeFeatures_Synteny.gff3 GFF3
  • source: NCBI - https://www.ncbi.nlm.nih.gov/.
    Created from the ref_GRCh38.p12_top_level.gff3 file from NCBI and contains the official gene symbol, gene name, and GeneID.
  • description: a standard GFF3 format file with the following Column 9 properties; 'ID', 'Symbol', 'Name', and 'Dbxref'. Note that the NCBI GeneID is stored in the 'Dbxref' field whereas the 'ID' field should be used only to identify parent/child features within the file.
    This file contains all of NCBI's genes and pseudogenes in a minimized format (all exons combined into a single feature)
Human_GeneAssociation_DO_Synteny.gaf GAF
  • source: Mouse Genome Informatics - http://www.informatics.jax.org/downloads/reports/MGI_DO.rpt
  • description: standard GAF format file in which the feature with annotation is stored in the second column and the specific ontology annotation term is stored in the fourth column.
Human_GeneAssociation_GO_Synteny.gaf GAF
  • source: source data is from a regularly generated report on the Gene Ontology Consortium's website and can be found here. The particular file to use is goa_human.gaf that is filtered to reduce complexity and circular annotations, which exist in other files.
  • description: standard GAF format file in which the feature with annotation is stored in the second column and the specific ontology annotation term is stored in the fourth column. File headers provide a more complete description.
Human_GWAS_Synteny.vcf VCF
  • source: the source data is from the EBI GWAS database, in particular, 1.0.2 all associations with added ontology annotations
  • description: contains GWAS loci that match the term 'type II diabetes mellitus'. Note that GWAS hits classified with multiple terms including 'type II diabetes mellitus' were not included. This file was converted into a standard VCF format with 4 additional fields: AF - allele frequency, CG - mapped gene symbols, LT - ontology term for loci, and PMID - the study's PubMed ID.
Human_CytoBand_Synteny.gff3 GFF3
  • source: UCSC - https://www.ucsc.edu/ - release 12 Jun, 2020
  • description: the original (UCSC) file has been converted into standard GFF3 format and renamed.
    Column 9 contains 3 attributes: 'ID' - the provided UCSC ID, 'Location' - the p/q arm position of this band, and 'Color' - the stain color of this band.

Rat Data

Data files from rat source data. For ontologies pertaining to rat see the General Data section.

Name File Format Details
Rat_GenomeFeatures_Synteny.gff3 GFF3
  • source: The Rat Genome Database - https://rgd.mcw.edu/rgdweb/homepage/ - release 12 Jun, 2020
  • description: standard GFF3 format file with the following Column 9 properties; 'ID', 'Symbol', 'Name', 'Dbxref', and 'Biotype'.
    The file contains all exons from the rn6 assembly for rat, which is the most current genome assembly.
Rat_QTL_Synteny.gff3.gz GFF3
  • source: The Rat Genome Database - https://rgd.mcw.edu/rgdweb/homepage/ - release 9 Jul, 2020
  • description: features in this file have the type QTL.
    The Column 9 field contains 3 attributes - 'ID' is the feature's RGD ID, 'Name' is the QTL's official name, and 'Dbxref' is RGD's ID again.
Rat_GeneAssociation_GO_Synteny.gaf GAF

  • description: standard GAF format file in which the feature with annotation is stored in the second column and the specific ontology annotation term is stored in the fourth column. File headers provide a more complete description.

Comparative Data

Name File Format Details
MouseHuman_Homologs_Synteny.tsv

RatHuman_Homologs_Synteny.tsv

RatMouse_Homologs_Synteny.tsv 		TSV
  • source: MouseMine - http://www.mousemine.org/mousemine/begin.do
  • description: a homolog is defined as two features, one from each - reference and comparison - genome, that correspond to each other. Each homolog is a single line of the file and includes its species ID, feature ID, feature symbol, feature chromosome, feature start, and feature end.
HumanRat_Blocks_Synteny.txt

RatMouse_Blocks_Synteny.txt

MouseHuman_Blocks_Synteny.txt 		TSV
  • source: MouseMine - http://www.mousemine.org/mousemine/begin.do
  • description: a syntenic block is defined as one region in a genome with a start, an end, and a chromosome that corresponds to another region in another genome also having a start, an end, and a chromosome. Each block will also have a strand value, either '+' or '-' that determines if the block's features run in the same direction or in the opposite directions. The last column requires an ID.

General Data

Name File Format Details
go-basic.obo OBO
  • source: The Gene Ontology Resource - http://geneontology.org/page/download-ontology - release 01 Jan, 2021
  • description: describes a gene/gene product in detail, considering three main aspects: its molecular function, the biological process in which it participates, and its cellular location
MPheno_OBO.ontology OBO
  • source: Mouse Genome Informatics - http://www.informatics.jax.org/downloads/reports/index.html - release 12 Jan, 2021
  • description: contains pre-coordinated phenotype terms, definitions and synonyms that can be used to describe mammalian phenotypes
HumanDO.obo OBO
  • source: The OBO Foundry - http://www.obofoundry.org/ontology/doid.html - release 22 Dec, 2020
  • description: describes the classification of human diseases organized by etiology
